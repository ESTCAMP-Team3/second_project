    ## ✅ 이전 실험 과정에서 알게 된 현실적 인사이트
    - Close-only 모델 → 초기 실험부터 drift copying 경향이 명확하게 나타남
    - 논문 재현 모델 → 로그수익률 기반으로 때때로 추세 변곡점을 예측하는 것처럼 보이지만, 실제로는 역산(로그 수익률 → 종가 계산 중 로그 누적곱)의 스무딩 오류일 가능성도 있음. 수익률 및 로그수익률은 랜덤워크가 심해 R²이 0 근처
    - 플래그 기반 조건부 신호 모델 → 단일 및 합성 신호 조건으로는 통계적으로 의미 있는 수익률 및 종가 예측 불가
    - 윈도우 길이 실험 → 3, 10, 30, 60일을 비교했을 때 Horizon=1 예측에서는 30일과 60일에서 윈도우가 가장 안정적인 R²를 보임
    - Buy and Hold 전략 비교 → 거의 항상 우위에 있어 모델 예측력의 한계를 확인

    - 입력 시퀀스 윈도우 길이: 60일

    ## ✅ 피처 생성 및 전처리
    - `ta` 라이브러리의 `add_all_ta_features` 함수를 사용해 **기술적 지표를 일괄 생성**
    - 생성된 지표 중 결측치가 한 달(약 40일) 이상 발생하는 컬럼 제거 → 데이터 안정성 확보
    - 결측치 제거 후 학습/검증 구간 분할
    - **XGBoost Regressor를 활용해 피처 선택**
    - 생성된 모든 기술적 지표를 사용해 내일 종가를 예측하는 모델을 학습
    - Feature Importance 기준 상위 8개 변수를 자동 선택
    - 종목별로 수동 선택 대신 머신러닝을 활용해 중요 변수 선별
    - 선택된 상위 피처들에 대해 MinMaxScaler로 스케일링
    - 타겟 종가도 MinMaxScaler로 스케일링
    - 시계열 윈도우 생성
    - 과거 60일치 피처를 입력
    - 다음날 종가를 출력 타겟으로 설정

    ---

    ## ✅ 사용 모델
    - Keras Sequential LSTM 모델
    - LSTM(64 units) → Dropout(0.2) → Dense(1)
    - 손실함수: MSE
    - Optimizer: Adam(learning rate=0.001)
    - Epochs: 100
    - Batch Size: 64
    - EarlyStopping 적용
    - patience=40
    - 최적 가중치 복원

    ---

    ## ✅ 모델 성능 향상법
    - 기술적 지표를 자동 생성하고 XGBoost 기반으로 타겟 종가와의 직접적인 중요도를 평가해 상위 변수를 자동 선택
    - 종목별 수동 선택 대신 머신러닝 기반 피처 엔지니어링
    - 불필요한 노이즈 변수 제거로 학습 안정화
    - 스케일링 방식 선택(MinMax/Robust 가능) → 실험에서는 MinMax 사용
    - EarlyStopping을 통한 과적합 방지
